Frequently Asked Questions

How to use the tool, how it works, and solutions to common problems.

How to use

Your order file must be in Excel format — .xlsx or .xls. CSV files are not supported.

The file should contain at minimum: product descriptions, pallet dimensions (length, width, height in cm), and an order quantity column. Weight data is optional but enables the per-container weight limit check.

  1. Choose file — click the file field and select your order spreadsheet.
  2. Quantity column — leave blank to use auto-detection. If results look wrong, enter the exact column header from your spreadsheet (e.g. External Packaging Quantity).
  3. Run — click Run Optimizer. Processing typically takes 10–60 seconds. The first request after a period of inactivity may take up to 50 seconds to start the server.
  4. Review — any warnings or issues appear directly on the page after the run completes.
  5. Download — click Download Packing Report to get the full Excel file with layout diagrams, container details, and fill recommendations.

The report has four sheets:

  • Overview — key metrics and a per-container fill chart.
  • Container Details — every pallet row and NP box placement per container, sorted by position, with dimensions, weight, and Y-start values.
  • Packing Layout — a colour-coded grid showing the physical layout of each container. Pallets and NP boxes appear on the same row at their correct positions. A green sub-row shows recommended additions.
  • Recommendations — additional items from your order that could be loaded into remaining free space, split by pallets and NP boxes per container.
How it works

Pallets are grouped by footprint and height into row blocks. Recognised footprints are 115×115 cm, 115×108 cm, 115×77 cm, and 77×77 cm (±2 cm tolerance). Heights are mapped to bands: <66 cm, 66–89 cm, 89–130 cm, >130 cm.

Row blocks are packed into containers using OR-Tools CP-SAT, a constraint programming solver. Tall blocks are loaded from the rear. Only the final (door-side) row must fit within the door height of 250 cm; all other rows may be taller.

NP (non-palletised) boxes fill the tail zone — the unused container length after the last pallet row. A column-based geometric algorithm stacks boxes across the full container width (235 cm) and up to door height, sharing columns across box types to avoid wasted gaps between sections.

NP boxes are never placed on top of pallets, keeping the loading sequence safe and straightforward.

After packing, the engine examines remaining free space in each container and suggests additional items from your current order. Two zones are analysed:

  • Tail zone — unused length after the last pallet row. Space is allocated proportionally across pallet types based on their order quantities. Remaining length after pallets is filled with NP boxes stacked to door height.
  • Atop zone — headroom above each placed pallet row (up to the door height). Pallet blocks only — boxes cannot be placed above pallets.

Containers that are already optimally packed show the remaining empty space with an explanation of why nothing more can be added from the current order.

Default dimensions: 1203 × 235 × 270 cm (length × width × height). Door opening height: 250 cm. Max payload weight: 18,000 kg. A 5 cm gap is reserved between pallet rows for fork-lift clearance.

These can be adjusted via environment variables on the server (CONTAINER_LENGTH_CM, CONTAINER_WIDTH_CM, etc.).

Common issues

Only .xlsx and .xls files are accepted. If you are working from a CSV or Google Sheet, export it as Excel first: File → Download → Microsoft Excel (.xlsx).

Pallets with footprints outside the recognised sizes (115×115, 115×108, 115×77, 77×77 cm ±2 cm) are skipped. A warning will appear on screen if this happens.

If the container count is higher or lower than expected, check the Packing Layout sheet to inspect fill patterns. Unexpectedly low fill on early containers often indicates a dimension mismatch or an order with very few items of a particular pallet type.

Auto-detection scans for common column names. If it selects the wrong column, enter the exact header text in the Quantity column field on the main page — including any spaces and capitalisation exactly as they appear in your spreadsheet.

Example: if your column is External Packaging Quantity, type that exactly.

Weight data is absent from your Excel file. The optimizer still runs correctly — packing and recommendations are unaffected — but the 18,000 kg per-container weight limit check is inactive. A notice will appear on the results page.

To enable weight checks, ensure your spreadsheet includes a weight column (kg per pallet).

Two things can cause this:

  • Cold start — the hosting service pauses after inactivity. The first request after a pause can take up to 50 seconds before the optimizer begins. Subsequent runs in the same session are fast.
  • Large or complex order — the solver runs up to 5 seconds per container. A 10-container order may take close to a minute. This is expected.

Use the Report a bug link at the bottom of the main page. Your message will be sent with the run summary and any warnings attached automatically — no need to copy-paste anything.
